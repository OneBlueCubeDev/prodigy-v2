   - **Duplicate Warning Banner** -- renders between Demographics and Guardian (positioned after section 1). Show only when `duplicates.length > 0`.
   - **Section 2: Guardian** -- Guardian Name* (Input), Guardian Phone (Input), Guardian Relation (Select with options: Parent, Grandparent, Aunt/Uncle, Sibling, Other)
   - **Section 3: Address & Phone** -- Address (Input, col-span-2), City (Input), State (Input), Zip (Input), Phone (Input)
   - **Section 4: SSN** -- SSN (Input type="text" inputMode="numeric" maxLength={11}, col-span-2). Strip dashes on form submit via transform: `value.replace(/\D/g, '')`.

3. Each section header uses `CollapsibleTrigger` with `ChevronDownIcon` from lucide-react. Section title uses `text-lg font-semibold`.

4. Form field layout per UI-SPEC: `grid grid-cols-1 md:grid-cols-2 gap-4` inside each section. Address and SSN span full width with `col-span-1 md:col-span-2`.

5. **Duplicate detection** (D-04, D-05):
   - Use `useWatch` to observe `firstName`, `lastName`, `dateOfBirth`, and compute SSN last 4 from the ssn field.
   - `useEffect` with 500ms debounce: when all three required fields are non-empty, call `checkDuplicate({ firstName, lastName, dateOfBirth, ssnLast4 })`.

   - On "Not a Match" dismiss: call `logDuplicateOverride` with the first duplicate's id and all matched ids, then clear duplicates state.

6. **Submit handler**:
   - Call `form.handleSubmit(onSubmit)` where `onSubmit` transforms SSN field (strip dashes), then calls `createYouth(data)`.
   - On success: `toast.success('Youth registered successfully.')` and `router.push('/youth/' + result.data.id)`.
   - On error: `toast.error('Registration failed. Please try again or contact support.')`.
   - Submit button: "Register Youth" primary variant, disabled during `isSubmitting`, shows "Registering..." text while submitting.

7. Page wrapper: `max-w-4xl mx-auto px-4 py-8 md:px-6`.

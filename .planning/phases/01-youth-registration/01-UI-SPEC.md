## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.25 |
| Display | 28px | 600 (semibold) | 1.2 |

Notes:
- Body maps to Tailwind `text-sm`. Used for form field values, table cell content, description text.
- Label maps to `text-sm`. Used for form field labels, column headers, nav items. Differentiated from body via uppercase tracking treatment where needed.
- Heading maps to `text-xl font-semibold`. Used for page title (e.g., "Register Youth", "Youth Records"), section headers within the detail view.
- Display maps to `text-2xl font-semibold`. Reserved for page-level h1 only (one per page).
- Uppercase tracking label (`text-xs uppercase tracking-wider`) used for section group labels (e.g., "Current Program"). Pattern established by Phase 0 sidebar. This treatment provides sufficient differentiation without a third weight.
- Exactly 2 weights in use: 400 (regular) and 600 (semibold).

## Color

All colors reference CSS custom properties from `src/app/globals.css`. No raw hex values in component code.

| Role | CSS Variable | Light Mode Value | Usage |
|------|-------------|-----------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) — white | Page background, form background |
| Secondary (30%) | `--secondary` / `--muted` | oklch(0.97 0 0) — near-white gray | Cards, sidebar (`bg-secondary/50`), table row alternates, muted sections |
| Accent (10%) | `--primary` | oklch(0.546 0.215 261.3) — blue-600 | See reserved list below |
| Warning | `--chart-2` / amber | amber-400 via `bg-yellow-400/10 border-yellow-400` | Duplicate warning banner only (D-06) |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) — red-600 | Destructive actions only |

Accent (`--primary`) reserved for:
- Primary CTA buttons only: "Register Youth" submit button, "Save Changes" in edit mode
- Active/focused ring on form inputs
- Pagination active page indicator
- Role badge variant `default` (Admin) — established by Phase 0 AppHeader

Accent is NOT used for:
- Nav links (use `text-foreground`)
- Section headings
- Read-only field values
- Table row hover (use `bg-muted/50`)

Warning color (yellow) reserved for:
- Duplicate detection banner (D-06) exclusively

Destructive color reserved for:
- No destructive actions in Phase 1 (delete is out of scope). Reserved in contract for future phases.

Dark mode: All values invert per `.dark` CSS class. No dark-mode-specific overrides needed in component code — CSS variables handle it.

## Layout

### Page Shell (pre-established by Phase 0)

- Sidebar: fixed 240px left, `bg-secondary/50`, hidden on mobile (Sheet overlay).
- Header: sticky 56px top bar, `bg-background border-b`.
- Content area: `flex-1 overflow-y-auto`. Pages render inside this area.
- Page content wrapper: `max-w-4xl mx-auto px-4 py-8 md:px-6` — constrains form and list content width.

### Youth List Page (`/youth`)

- Top row: Page title (h1 Display), "Register Youth" button (right-aligned).
- Search bar: Full-width `Input` with search icon, below the title row, `mb-6`.
- Results: shadcn `Table` component. Columns: Name, Date of Birth, SSN (masked), Registered.
- Empty state: centered within the table area (see Copywriting Contract).
- Pagination: below the table, centered. "Previous / 1 2 3 / Next" pattern (D-09).
- Default view: most recently registered youth, `created_at` descending (D-08).

### Youth Registration Form (`/youth/new`)

- Single scrollable page. No tabs, no wizard (D-01).
- Page title: "Register Youth" (h1 Display).
- Form sections use shadcn `Collapsible`. Four sections in order:
  1. Demographics (First Name*, Last Name*, Date of Birth*, Gender, Race, Ethnicity)
  2. Guardian (Guardian Name*, Guardian Phone, Guardian Relation)
  3. Address & Phone (Address, City, State, Zip, Phone)
  4. SSN
- All sections default to open on initial render (open question resolved: staff needs all fields visible for new registration).
- Duplicate warning banner (D-06): renders between the Demographics section and the Guardian section. Yellow `Alert` component. Hidden until a duplicate is detected.
- Form field layout: 2-column grid on `md+` breakpoint (`grid grid-cols-2 gap-4`), single column on mobile (`grid-cols-1`). Exceptions: Address field spans full width (`col-span-2`). SSN field spans full width.
- Action row: bottom of form. "Register Youth" primary button (right). No cancel — use browser back.

### Youth Detail / Edit Page (`/youth/[youthId]`)

- Read-only default. "Edit Youth" button in top-right (D-10).
- Content in shadcn `Card` components. One Card per section (Demographics, Guardian, Address & Phone).
- SSN card: shows masked value for non-admin roles; decrypted value for Admin (D-03).
- In edit mode: Card content switches from static text to form inputs. "Save Changes" (primary) and "Discard Changes" (ghost) buttons in action row at bottom of each card or at page bottom.
- No stub sections for enrollment or attendance (D-11).
- No audit trail section (D-12).

## Interaction Patterns

### Duplicate Detection (D-04, D-05, D-06)

- Trigger: fires after `firstName`, `lastName`, and `dateOfBirth` are all non-empty.
- Debounce: 500ms on field change.
- Refines: if `ssnLast4` is entered later, re-fires the check.
- Race condition guard: `latestRequestId` ref pattern — only apply results if request ID matches latest call.
- Banner: yellow `Alert` with `variant="warning"` styling (`bg-yellow-50 border-yellow-400 text-yellow-900`). Positioned between Demographics and Guardian sections.
- Banner content: "Possible duplicate found — [Name], born [DOB]." + "Not a Match" ghost button.
- "Not a Match" dismisses the banner and logs override to audit trail via Server Action.
- Register button remains enabled during and after duplicate detection (D-06 — not a blocker).

### Search (D-07, D-08, D-09)

- Debounce: 300ms on search input change.
- Implementation: `useSearchParams` + `router.push` to update `?q=` URL param.
- Server Component reads `?q` and `?page` from `searchParams` prop, queries Prisma.
- Pagination: numbered page buttons. Active page uses `bg-primary text-primary-foreground`. Previous/Next use ghost variant.
- Loading state: Skeleton rows (5 rows) shown while navigation is pending (use `useTransition` or Next.js loading.tsx).

### Inline Edit Toggle (D-10)

- "Edit Youth" button in detail page header triggers `setIsEditing(true)` via `useState`.
- All three section cards render input fields in edit mode.
- "Save Changes" calls `updateYouth` Server Action. On success: `setIsEditing(false)`, success toast, `revalidatePath` fires from server action.
- "Discard Changes" calls `setIsEditing(false)`, resets form to original values via `form.reset(mapYouthToFormValues(youth))`.
- Stale data prevention: `revalidatePath('/youth/[youthId]', 'page')` inside `updateYouth` Server Action.

### SSN Field

- Input type: `text`, `inputMode="numeric"`, `maxLength={11}` (formatted: XXX-XX-XXXX).
- Display in registration: visible to all roles during entry.
- Display in detail (read-only): `***-**-{last4}` for non-admin; decrypted for admin (D-03).
- Strip formatting before schema validation: remove dashes, validate 9 digits via Zod.

### Date of Birth Field

- Input type: `date` (native browser date picker). No custom DatePicker component needed.
- Storage: `@db.Date` in Prisma — date only, no time component.
- Display: format via `Intl.DateTimeFormat('en-US', { timeZone: 'UTC', month: '2-digit', day: '2-digit', year: 'numeric' })` to prevent timezone drift.

### Loading States

- List page: 5 Skeleton rows matching table column widths while search results load.
- Detail page: full-page skeleton with 3 Card-shaped skeletons.
- Form submit: Button disabled + loading spinner text ("Registering..." / "Saving...") while Server Action is in flight. Use `form.formState.isSubmitting`.

## Accessibility

- All form inputs use shadcn `FormField` + `FormLabel` + `FormMessage` — auto-associates label with input via `aria-describedby`.
- Collapsible section triggers: include `aria-expanded` and `aria-controls` (shadcn Collapsible handles this).
- Duplicate warning banner: `role="alert"` on the Alert component so screen readers announce it immediately on appearance.
- "Not a Match" button: `aria-label="Dismiss duplicate warning and continue registration"`.
- Search input: `aria-label="Search youth by name, date of birth, or SSN last 4"`.
- Pagination: `aria-label="Pagination"` on nav wrapper. Active page button: `aria-current="page"`.
- Touch targets: all interactive elements minimum 44px height (`h-11` on inputs/buttons).
- Focus ring: `outline-ring/50` applied globally via `globals.css` `@layer base`.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default element spacing, form field gaps |
| lg | 24px | Section padding, card internal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical rhythm |
| 3xl | 64px | Auth page centering offset |

Exceptions:
- Touch targets on program selector cards: minimum 44px height (accessibility requirement)
- Clerk-hosted login UI: spacing is Clerk-controlled, not subject to this contract

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 600 (semibold) | 1.2 |

**Rationale:** Internal tool used by staff at desks and on mobile. 14px body is standard for dense data interfaces. 20px heading sufficient for section hierarchy. Display (28px) reserved for page-level titles only (e.g., "Select a Program"). Two weights only: 400 and 600.

**Font stack:** Inter via `next/font/google`. Fallback: `system-ui, sans-serif`.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `hsl(0 0% 100%)` — white | Page background, form surfaces |
| Secondary (30%) | `hsl(0 0% 96.1%)` — zinc-100 | Sidebar background, card surfaces, nav |
| Accent (10%) | `hsl(221.2 83.2% 53.3%)` — blue-600 | Primary CTA buttons, selected program card border, active nav item indicator |
| Destructive | `hsl(0 72.2% 50.6%)` — red-600 | Destructive action buttons only |

**Source:** shadcn default neutral palette. Specific hex values match shadcn CSS variable defaults at initialization.

Accent reserved for:
1. "Sign In" button (Clerk redirect trigger — if a custom button is needed)
2. Primary CTA on program selector ("Select Program" / "Continue" button)
3. Active/selected state border on program selector cards
4. Active nav item left-border indicator in app shell sidebar

Accent is NOT used for: informational text, hover states on non-CTA elements, icons, headings.

Dark mode: Not in scope for Phase 0. shadcn ships with dark mode CSS variables — do not activate. Single light mode only.

## App Shell Layout Contract

The app shell is the outermost layout wrapping all authenticated routes. Phase 0 establishes it as the inherited foundation for all subsequent phases.

**Structure:**
```
[Root Layout]
  └── [Auth Layout — /sign-in, /sign-up]   (Clerk, no shell)
  └── [App Layout — (app)]
        ├── Sidebar (fixed left, 240px wide)
        │     ├── Logo + app name (top)
        │     ├── Nav items (icon + label)
        │     └── User info + sign-out (bottom)
        └── Main content area (flex-1, scrollable)
              ├── Page header (sticky, 56px height)
              └── Page content (16px padding)
```

**Sidebar widths:**
- Desktop: 240px fixed
- Mobile: hidden (hamburger toggles slide-out sheet overlay)

**Header height:** 56px (14px on 4px scale)

**Nav items in Phase 0 sidebar** (placeholders — expand in later phases):
- Dashboard (home icon) — placeholder route `/`
- Program indicator (current selected program name, not a nav link)
